# 技術決策記錄 (Architecture Decision Records)

## 1. 使用 Go 而非 Python 或 Laravel

**日期**: 2026-03-11
**狀態**: 待決

### 背景

PRD 提到可使用 Python 或 Go，而環境也支援 Laravel (PHP)。

### 決策

選擇 **Go 1.23** 作為主要語言。

### 理由

| 因素 | Go | Python | Laravel |
| :--- | :--- | :--- | :--- |
| Deterministic | 高 (靜態型別) | 中 (動態型別) | 中 (靜態型別) |
| 執行效能 | 高 | 中 | 中 |
| CLI 部署 | 單一執行檔 | 需解釋器 | 需 PHP 環境 |
| 並行處理 | Goroutine 輕量 | 需 asyncio | 需協程 |
| 開發速度 | 中 | 高 | 高 |
| 生態系 | 完整 | 強大 | PHP 專屬 |

### 益處

1. **單一执行檔**：使用者只需下載一個 binary 即可運行
2. **Docker 友好**：透過 multi-stage build 可得到極小鏡像
3. **穩定性**：靜態型別在數值計算時更安全

### 潛在風險

- 開發速度較 Python 慢
- 需要學習 Go 生態系

### 監控指標

- 編譯時間
- 二進位檔案大小
- CLI 響應時間

## 2. Clean Architecture 目錄結構

**日期**: 2026-03-11
**狀態**: 已決定

### 決策

採用標準的 Go Clean Architecture 目錄結構。

### 理由

1. **清晰職責**：cmd/internal/pkg/tests 各司其職
2. **易於測試**：內部模組可獨立測試
3. **擴充性**：未來可輕鬆新增 Web API 或 GUI

## 3. 八字計算使用純數學演算法

**日期**: 2026-03-11
**狀態**: 已決定

### 決策

八字計算不依賴外部 API 或資料庫，全部使用數學公式計算。

### 理由

1. **Deterministic**：相同輸入必然得到相同輸出
2. **可用性**：無需網絡連線即可運行
3. **效能**：無 I/O 等待時間

### 實作要点

- 需要建立完整的干支對照表
- 需要 implementing 節氣計算公式
- 需要處理閏月邏輯

**日期**: 2026-03-11
**狀態**: 待決

### 決策

LLM 整合使用介面抽象，支援多種提供商。

### 結構

pkg/llm/
├── api.go       # 通用 API 介面
├── openai.go    # OpenAI 實作

### 理由

## 5. 人格生成使用映射表而非 ML

**日期**: 2026-03-11
**狀態**: 已決定

### 決策

人格特徵來自預定義的映射表，而非機器學習模型。

### 理由

1. **可解釋性**：清楚知道每個特徵的來源
2. **控制性**：可精確調整文化特定內容
3. **零依賴**：無需訓練模型或下載权重

### 映射表範例

```
十神 + 五行 = 特徵
正官 + 木 = 正直、有原則
七殺 + 火 = 果斷、勇猛
偏财 + 土 = 節儉、務實
```
