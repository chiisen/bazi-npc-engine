## 1. 🎯 專案特色 (Project Profile)

### 基本資訊

| 項目 | 內容 |
| :--- | :--- |
| 專案名稱 | Bazi NPC Generator |
| 技術棧 | Go 1.23 + Clean Architecture |
| 作業系統 | macOS |
| 目標 | 建立基于八字的人格生成引擎 |

### 專案簡介

**Bazi NPC Generator** 是一個使用「八字（Bazi）」作為角色種子 (seed) 的 RPG NPC 生成系統。

```
出生時間 → 八字 → 人格模型 → NPC Persona → LLM 對話 / 行為
```

### 核心理念

- **Deterministic Engine**：相同輸入必然得到相同輸出
- **可解釋性**：清楚知道每個特徵的來源
- **零依賴**：無需網絡連線即可運行

## 4. 代碼規格與開發習慣 (Coding Standards)

### Go 規範

- **語法風格**：遵循 Go 官方 `gofmt` 規範。
- **命名規範**：
  - Package：小寫單詞，簡短有意義
  - 函式/變數：CamelCase，公開 使用 PascalCase
  - 常數：大寫 SNAKE_CASE
- **註解規範**：
  - 每個公開函式上方必須附上 **繁體中文** 註解
  - 複雜邏輯需添加行內註解
  - 使用 `// ═══════════════════════════════════════════` 分隔註解區塊

### 要求

- 遵循 Clean Architecture 目錄結構
- 核心邏輯與外部依賴分離
- 每個模組應有單元測試

### 測試規範

- 單元測試檔案以 `_test.go` 結尾
- 使用 `testing` 標準套件
- 測試檔案結構：`tests/unit/`

## 5. 目錄結構規範 (Directory Structure)

```
bazi-npc-engine/
├── cmd/              # CLI 入口 (main 函式)
├── internal/         # 專業邏輯 (不對外公開)
├── pkg/              # 公開ライブラリ
├── tests/            # 測試檔案
├── docs/             # 文檔
│   ├── PRD.md        # 產品需求文件
│   ├── ARCHITECTURE.md
│   ├── API.md
│   ├── DIRECTORY.md
│   └── DECISIONS.md
├── Makefile
├── go.mod
└── CHANGELOG.md
```

### 目錄職責

| 目錄 | 職責 |
| :--- | :--- |
| `cmd/` | 應用程式入口點，每個子目錄代表一個可執行檔 |
| `internal/` | 專案核心邏輯，package 不會被其他專案 import |
| `pkg/` | 公開 API，可被其他專案引用 |
| `tests/` | 所有測試檔案 |
| `docs/` | 專案技術文檔 |

## 7. Git 提交規範 (Git Commit Standards)

- **格式結構**：`<type>(<scope>): <subject>`
  - 第一行：簡短標題，不超過 50 字元
  - 第二行：空行
  - 第三行：詳細描述（每行不超過 72 字元）

### Type 類型

- `✨ feat` (新功能)
- `🐛 fix` (修正)
- `📝 docs` (文件)
- `💄 style` (格式)
- `♻️ refactor` (重構)
- `🧪 test` (測試相關)
- `🔧 chore` (雜項)
- `⏪️ revert` (回滾)

### Subject 主旨

使用 **繁體中文**，簡短描述變更。

### Body 內容

詳細描述變更內容、原因及背景，使用 **繁體中文**。
